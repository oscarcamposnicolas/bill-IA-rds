¿Modelo Ciego o Inseguro?

Análisis avanzado de errores para diagnosticar la causa raíz de los fallos, distinguiendo entre objetos que el modelo no ve y aquellos sobre los que duda, para guiar la mejora final del dataset.

El Diagnóstico Final: Más Allá de las Métricas

Desafío

El modelo `best.pt` aún falla en imágenes reales. El simple hecho de saber que falla (Falso Negativo) no es suficiente para saber cómo solucionarlo. ¿El modelo es "ciego" o simplemente "inseguro"?

Propuesta

Realizar un análisis diferencial ejecutando las pruebas con un umbral de confianza muy bajo (`conf=0.25`) para revelar las detecciones de las que el modelo "duda" pero que sí está viendo.

Acciones

  • Ejecución de inferencia con `conf=0.25`.
  • Análisis visual comparativo de resultados.
  • Clasificación de errores: "ciego" vs. "inseguro".
  • Creación de una "lista de la compra" para mejorar datos.

Resultados

Diagnóstico definitivo: el problema principal no es de "ceguera", sino de "inseguridad" debido al "Domain Gap". Esto permite un plan de acción quirúrgico para la mejora final del dataset.

Análisis Diferencial de Confianza

La Técnica de Diagnóstico

Se ejecuta el modelo `best.pt` sobre las imágenes de prueba con un umbral de confianza muy bajo. Esto nos permite separar los errores en dos categorías:

  • Modelo Ciego: Bolas que siguen sin detectarse. Requieren añadir más diversidad al dataset (nuevos ángulos, luces).
  • Modelo Inseguro: Bolas que ahora sí aparecen, pero con baja confianza. Se pueden solucionar añadiendo más ejemplos de esas situaciones específicas.

Conocimientos y Conceptos Aplicados

Análisis de Errores Cualitativo

Ir más allá de las métricas para entender el *porqué* de los fallos (oclusiones, reflejos, etc.), lo que permite una mejora de datos dirigida y eficiente.

Curación de Dataset 2.0

La recolección de nuevos datos no es aleatoria, sino que se basa en un "informe de errores" para atacar específicamente las debilidades del modelo.

Compromiso Precisión-Recall

El ajuste del umbral de confianza (`conf`) demuestra este equilibrio: un umbral bajo aumenta el recall (más detecciones) pero baja la precisión (más errores), y viceversa.

Falsos Negativos por Umbral

El descubrimiento más importante: objetos que el modelo sí ve, pero con confianza insuficiente. Son el objetivo principal para mejorar en el re-entrenamiento.

Robustez del Modelo

El objetivo final de esta fase: hacer que el modelo no solo sea preciso en su "zona de confort" (el dataset), sino también robusto en el impredecible mundo real.

Fine-Tuning Enfocado

La estrategia de re-entrenamiento final: usar el `best.pt` como base y re-ajustarlo con el nuevo dataset enriquecido para que aprenda a manejar los casos difíciles.

Glosario de Términos

Modelo Ciego
Un falso negativo absoluto. El modelo es incapaz de proponer una caja delimitadora para un objeto, incluso con una confianza extremadamente baja. Indica un problema fundamental en los datos.
Modelo Inseguro
Un falso negativo por umbral. El modelo sí detecta el objeto, pero le asigna una puntuación de confianza tan baja que es filtrado. Es un problema más fácil de solucionar.
Análisis Diferencial
Técnica de comparar los resultados de un modelo bajo dos condiciones diferentes (en este caso, dos umbrales de confianza) para diagnosticar la causa raíz de un problema.
Oclusión
Situación en la que un objeto está parcialmente oculto por otro, dificultando su detección.

Conclusiones y Siguientes Pasos

Conclusiones Clave

  • El análisis de baja confianza es una herramienta de depuración potentísima que ha permitido diagnosticar con precisión que el problema es de "inseguridad" y no de "ceguera".
  • El "Domain Gap" entre el dataset y las imágenes reales es la causa principal de la baja confianza del modelo.
  • Se ha creado una "lista de la compra" de datos muy específica, basada en evidencia, para la mejora final y más importante del modelo.

La Misión Final

1. Campaña de Datos 3.0

Recolectar y etiquetar imágenes que cubran las debilidades detectadas: replicar condiciones de la "Imagen 5", añadir oclusiones y resolver confusiones de clase.

2. Re-entrenamiento Definitivo

Realizar un último `fine-tuning` partiendo del `best.pt` actual y usando el nuevo dataset enriquecido para crear el modelo de producción final.

3. Despliegue y Validación Final

Probar el modelo final en un amplio espectro de vídeos e imágenes en tiempo real para validar que la brecha de dominio ha sido cerrada con éxito.
