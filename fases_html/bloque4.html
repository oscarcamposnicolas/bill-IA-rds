Aumentando la Capacidad del Modelo

Exploramos los límites del rendimiento cambiando a una arquitectura superior (YOLOv11m) y descubrimos la importancia de la brecha de generalización entre los datos de entrenamiento y el mundo real.

El Límite del Modelo y la Brecha con la Realidad

Desafío

El modelo anterior (YOLOv8n), aunque optimizado, alcanzó un techo de rendimiento. Se planteó la hipótesis de que la propia capacidad del modelo era el factor limitante.

Propuesta

Realizar un nuevo entrenamiento comparativo utilizando un modelo de mayor capacidad (YOLOv11m), manteniendo la misma configuración de datos y aumentación para una evaluación justa.

Acciones

  • Cambio de modelo base a `yolo11m.pt`.
  • Ajuste de `batch_size` a 24 para gestionar la VRAM.
  • Entrenamiento en GPU local (ROCm).
  • Análisis comparativo de `best.pt` vs. `last.pt`.

Resultados

El modelo `medium` superó claramente al `nano`, alcanzando un mAP50 de ~0.80. Se identificó una brecha de generalización ("Domain Gap") entre los datos del dataset y las imágenes del mundo real.

Análisis Comparativo: `best.pt` vs `last.pt`

La Trampa del Sobreajuste: `last.pt`

El modelo de la última época (`last.pt`) está altamente especializado en los datos de entrenamiento. Al probarlo en imágenes nuevas del "mundo real", su rendimiento decae drásticamente, ya que ha "memorizado" en lugar de "aprender a generalizar".

Características de `last.pt`

  • Excelente rendimiento en datos del mismo dominio.
  • Bajo rendimiento en datos de dominios diferentes.
  • Propenso a ignorar objetos en condiciones de luz o ángulos no vistos.

El Modelo Campeón: `best.pt`

YOLO guarda automáticamente el modelo que obtiene el mejor rendimiento en el set de validación (`best.pt`). Este modelo, proveniente de una época anterior (época 26 en este caso), está mucho menos sobreajustado y demuestra una capacidad de generalización muy superior, siendo la elección correcta para producción.

Características de `best.pt`

  • Mejor equilibrio entre Precisión y Recall.
  • Mayor capacidad para detectar objetos en imágenes nuevas.
  • Representa el punto óptimo del aprendizaje del modelo.

Conocimientos y Conceptos Aplicados

Capacidad del Modelo

Un modelo más grande (más parámetros) tiene un potencial de aprendizaje superior, pero también es más propenso al sobreajuste y requiere más recursos computacionales.

Brecha de Dominio (Domain Gap)

La diferencia de rendimiento que sufre un modelo cuando se enfrenta a datos de un "dominio" (ej. fotos reales) diferente al dominio con el que fue entrenado (ej. dataset curado).

Análisis Cualitativo

Inspección visual de los resultados del modelo para entender *por qué* falla (oclusiones, reflejos, etc.), más allá de las métricas numéricas, para guiar la mejora de los datos.

Ajuste Fino (Fine-Tuning)

Proceso de tomar un modelo ya entrenado (`best.pt`) y continuar su entrenamiento con un dataset nuevo o enriquecido para mejorar su rendimiento en casos específicos.

Curva de Aprendizaje

Representación gráfica de la mejora del modelo a lo largo de las épocas, observando la evolución de las métricas de pérdida (loss) y precisión (mAP).

Control de Potencia de GPU

Uso de herramientas como CoreCtrl para gestionar la temperatura y el consumo de la GPU durante entrenamientos largos, asegurando la estabilidad del hardware.

Glosario de Términos

Modelo `medium`
Una versión de la arquitectura YOLO con mayor capacidad (más parámetros) que los modelos `nano` o `small`, diseñada para ofrecer mayor precisión a cambio de un mayor coste computacional.
`best.pt`
El archivo de pesos guardado por Ultralytics que corresponde a la época con el mejor rendimiento en el conjunto de datos de validación. Es el modelo ideal para producción.
`last.pt`
El archivo de pesos que corresponde a la última época completada del entrenamiento. A menudo está más sobreajustado que `best.pt`.
Inferencia
El proceso de utilizar un modelo ya entrenado para hacer predicciones sobre datos nuevos que no ha visto antes.

Conclusiones y Siguientes Pasos

Conclusiones Clave

  • Un modelo de mayor capacidad (YOLOv11m) superó objetivamente al modelo `nano`, validando la hipótesis de que el tamaño del modelo era un factor limitante.
  • El análisis de la brecha de dominio es fundamental: el rendimiento de un modelo puede variar drásticamente entre datos del dataset y datos del mundo real.
  • La solución para mejorar la robustez del modelo ya no reside en ajustar hiperparámetros, sino en enriquecer el dataset con imágenes que cubran los casos de fallo detectados.

Próximos Pasos

1. Cerrar la Brecha de Dominio

Enfocarse en añadir al dataset imágenes "reales", especialmente con características similares a las de las pruebas donde el modelo falló (ej. `test_pool_table_5`).

2. Abordar Confusiones de Clase

Añadir ejemplos de primer plano y en diferentes iluminaciones de las bolas que el modelo confunde, como las lisas y rayadas del mismo color (`purple_4` vs `purple_12`).

3. Fine-Tuning Final

Realizar un último ciclo de entrenamiento (fine-tuning) partiendo del `best.pt` actual, pero con el nuevo y mejorado conjunto de datos.
