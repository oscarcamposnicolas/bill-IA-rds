Fase 10: Aplicación Web de Presentación

Culminando el proyecto con la creación de una aplicación web interactiva usando Flask para demostrar el sistema de IA, permitiendo la carga de imágenes y la visualización de resultados en tiempo real.

De la Consola a la Interfaz Gráfica

Desafío

Un modelo de IA, por muy potente que sea, tiene un impacto limitado si no se puede interactuar con él. Se necesita una interfaz para que los usuarios puedan probarlo y ver los resultados.

Propuesta

Desarrollar una aplicación web completa con Flask, un micro-framework de Python, que sirva de portfolio interactivo para todo el proyecto.

Acciones

  • Estructuración del proyecto web (carpetas `static`, `templates`).
  • Refactorización del código de IA a un módulo (`motor_inferencia.py`).
  • Creación del servidor web con `app.py`.
  • Diseño de la interfaz de usuario con HTML, CSS y JS.

Resultados

Una aplicación web funcional y profesional que sirve de portfolio multimedia, permite probar el modelo en tiempo real e incluye toda la documentación del proyecto.

Refactorización: Separando la Lógica de la Presentación

Backend: `app.py` y el Motor de IA

Frontend: HTML, CSS y JavaScript

Se crea una interfaz de usuario interactiva. El archivo `inferencia.html` contiene los elementos para subir la imagen. El `script.js` se encarga de enviar la imagen al servidor mediante una petición `fetch` a la API `/predict` sin recargar la página, y de mostrar el resultado dinámicamente.

script.js (Lógica del Navegador)

Conocimientos y Conceptos Aplicados

Flask

Micro-framework de Python utilizado para construir el servidor web de forma rápida y sencilla, gestionando rutas y peticiones HTTP.

Arquitectura Cliente-Servidor

Diseño fundamental de la web, donde el navegador (cliente) solicita información a un servidor, que la procesa y devuelve una respuesta.

Refactorización de Código

Práctica de reestructurar el código en módulos lógicos (ej. `app.py` vs `motor_inferencia.py`) para mejorar su legibilidad, mantenibilidad y reutilización.

API REST

Creación de un endpoint (`/predict`) que acepta peticiones POST para realizar una acción específica (la inferencia) y devolver un resultado en formato JSON.

JavaScript Asíncrono (Fetch API)

Uso de JavaScript para comunicarse con el servidor en segundo plano, permitiendo una experiencia de usuario dinámica sin necesidad de recargar la página.

Plantillas Jinja2

Sistema de plantillas de Flask que permite crear un `base.html` y extenderlo para reutilizar código HTML común (cabecera, navegación) en todas las páginas.

Conclusión del Proyecto

Lecciones Finales

  • La creación de una interfaz de usuario no es un paso final, sino una parte integral del desarrollo de IA que permite la validación, depuración y demostración del valor del proyecto.
  • Una buena arquitectura de software (separando backend y frontend, y la lógica de la aplicación de la lógica de IA) es fundamental para crear sistemas escalables y mantenibles.
